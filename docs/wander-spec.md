# 走りまくりモード（ツーリングループ生成）仕様ドラフト

目的地に着くことより、いい道（峠・展望）を長く走るのが目的。目標距離に近い周回 or 片道を自動生成する。

## 決定事項（合意済み）

- 形: 周回（自宅発着）と 片道（距離稼ぎ）の両対応。
- 寄るスポット: 峠・ワインディング（mountain_pass=yes）と 展望・景観（tourism=viewpoint）。
- 距離: 目標距離を指定（--dist KM）。
- リスト表示: 左袖のまま（ポップアップ化しない）。

## 入力

- 起点: --home / 現在中心 / --here
- 形: shape = loop | oneway
- 目標距離: --dist KM
- スポット種別: 既定 pass,view（峠・展望）
- 方角（oneway時・任意）: --bearing DEG（省略でランダム）

## 生成アルゴリズム（案）

1. 探索半径 R を目標距離から見積もる。loop は R ≈ dist/5〜dist/(2π)、oneway は R ≈ dist/2。
2. 起点中心・半径 R でスポットを Overpass 収集（峠・展望）。
3. スポットを選んで順序化し目標距離へ寄せる。
   - loop: 起点から放射状に散るよう数個選び、方位角順に並べて起点へ戻す（自己交差を減らす）。
   - oneway: 目標方角の扇形にあるスポットを距離順に数個。
4. 点列を BRouter で下道（moped）ルーティング。
5. 実距離が目標から外れたらスポット数を ±して数回リトライ（目標 ±20% を目安）。ランダム要素で毎回変える。

## 出力・操作

- 生成ルートを通常のルート状態に載せる（水色線＋スポットマーカー・距離/時間表示）。
- CLI: `termmap --wander --home LAT,LON --dist 60 --shape loop`
- 対話: `W` で生成。連打で別案（ランダム）。生成後は 保存(S)/共有(o)/GPX(g) が使える。

## 正直な制約

- スポット密度に依存（山方面は峠豊富・都心は少ない → 展望で補う。0件なら生成失敗を明示）。
- 目標距離ぴったりは不可。±20% 目安、数回リトライで寄せる。
- 「曲がり最大化」「水沿い選び」はこのモードでは扱わない（BRouter 独自プロファイル領域・別ピース）。
- 生成で Overpass/BRouter を数回叩く（時間がかかる）。

## 段階リリース

1. loop 生成（峠・展望・目標距離・ランダム）＋ CLI `--wander`。
2. oneway。
3. 対話キー `W`（連打で別案）。
4. スポット追加（道の駅・公園）・方角指定。
