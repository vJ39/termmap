# 動き・標高系機能 仕様ドラフト

3D地図プロダクト発想の翻訳。terminalで3Dは狙わず、標高プロファイル・ルート再生・ライブ現在地の3つを入れる。

## 確定事実（実データで確認済み）

- ルーティング = BRouter。geojson の座標は `[lon, lat, 標高(m)]` の3次元。標高は既存のルート取得にそのまま乗る（現状は3つ目を破棄している）。props に `filtered ascend`（累積登坂）, `plain-ascend`, `track-length`, `total-time`。
- 現在地 = `gps_here()` が `CoreLocationCLI` を1回叩く方式（要 `brew install corelocationcli` ＋ Location Services 許可）。ライブは背景スレッドで N秒ごとに呼ぶ。
- `roadtrace.rs`（新規）に polyline の累積距離歩行・補間・サンプリングがある。再生の位置補間に再利用。
- 対話ループは `event::poll` の非ブロッキング。背景処理は mpsc channel（ルート計算と同じパターン）。

## 共有データ層の変更（route.rs）

- `RouteResult` に `ele: Vec<f64>`（pts と同数）と `ascend_m: f64` を追加。`fetch_route` が coord[2] と `filtered ascend` を拾う。既存挙動は不変（追加のみ）。3機能すべての土台。

## 1. 標高プロファイル

- キー `E` トグル。下部ストリップ（高さ ~10行）に distance×elevation の折れ線を braille で描く（既存 braille を再利用）。列数=端末幅に合わせて距離でビン化。
- 表示: 最低/最高標高、累積登坂（ascend_m）、現在の勾配。局所最大（峠）をマーク。
- 任意（後）: プロファイル上のカーソル位置を地図側にハイライト。
- 既定: 配置=下ストリップ / 高さ10行 / braille。

## 2. ルート再生アニメ

- キー `A` で再生/一時停止（要 route）。`[` `]` で速度、`0` で先頭へ。
- 状態: `playback { dist_m, speed_mps, playing }`。dist を進めて roadtrace 補間で marker の lat/lon を出す。自動パンで中心追従。末尾で停止（ループしない）。
- 速度: 既定は「全ルートを ~20秒で走る」正規化。`[` `]` で 0.5x〜4x。現在の速度と進捗% を表示。
- タイルstutter対策: 再生前にルート沿いの必要タイルをキャッシュへプリロード（「プリロード中」表示）→ 再生はキャッシュから滑らかに。
- marker: 専用マーカー＋通過跡を薄く残す（任意）。
- 既定: キー=A / 速度=全体20秒正規化+可変 / プリロードあり / 自動パンあり / 末尾停止。

## 3. ライブ現在地（ブレッドクラム）

- CLI `--here-live`、対話キー `G` トグル。背景スレッドが `gps_here()` を N秒（既定5s）ごとに呼び、(lat,lon) を channel で送る。
- 表示: 自位置マーカー＋通過点の薄いトレイル（ブレッドクラム）。任意で自位置に自動センタリング（トグル）。
- 保存: トレイルはメモリのみ。任意で GPX 書き出し（既存エクスポート再利用）。
- 測位不可時: マーカー据え置き＋「測位待ち」表示。
- 既定: 間隔=5s / 自動センタリングはトグル / トレイルはメモリ+任意GPX。

## 実装フェーズ（分割リファクタ完了後・Phase B の後半）

1. route.rs: RouteResult に ele/ascend 追加 ＋ fetch_route 取得（小・全機能の土台）
2. 標高プロファイル（E）
3. ルート再生（A）＋タイルプリロード
4. ライブ現在地（G / --here-live）

## 正直な制約

- 実3D/GPU描画はやらない（terminal は2Dテキスト）。
- 再生の自動パンはタイル取得を伴う。プリロードしても未キャッシュ域では一瞬待つ。
- ライブ現在地は「走行中に terminal を見る」用途は限定的。計画確認/テスト/同乗/スマホ端末向け。
- 標高は SRTM 由来（BRouter）。数m単位の誤差あり。勾配は目安。
