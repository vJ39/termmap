# マイスポット 仕様ドラフト

ラーメン等のお気に入り地点を、カテゴリ別に色分けして地図に重畳する。表示/非表示を切り替えられる。
既存の「お気に入りルート(S/L)」「目的地カテゴリ検索(f)」とは別。

## 決定事項（合意済み）

- カテゴリは自由入力。カテゴリの 新規 / 更新(改名・色変更) / 削除 ができる。
- カテゴリ削除は、そのカテゴリに属すスポットが 0 件のときだけ許可（先に空にする）。
- 表示/非表示は全体トグル（1キーで全部 on/off）。

## データ

- カテゴリ: `~/.config/termmap/spot-categories.txt` … 1行 = `カテゴリ名<TAB>色index`
- スポット: `~/.config/termmap/spots.txt` … 1行 = `lat,lon,カテゴリ名,スポット名`
- 色: 高彩度パレット(10色程度)。カテゴリ作成時に空いている色を自動割当、後で変更可。色は index で保持し改名/並び替えでも安定。

## 操作（対話モード）

トップは `P`（マイスポット）。押すとメニュー:
- `a` 追加: 中心クロスヘアの位置にスポットを置く → カテゴリ選択(既存一覧 or 新規) → 名前入力
- `l` 一覧: マイスポット一覧(左袖) → ↑↓選択 / Enter その地点へ移動 / x 削除 / Esc 閉
- `k` カテゴリ管理: 一覧(左袖) → n 新規 / r 改名 / c 色変更 / x 削除(空のときのみ) / Esc 閉
- `v` 表示トグル: マイスポットの重畳を on/off
- `Esc` 閉じる

地図表示: マイスポットをカテゴリ色のマーカーで重畳（表示ON時）。ルートやリングと同じオーバーレイ層に載せる。

## CLI（任意・後回し可）

- `--spots-add "lat,lon,カテゴリ,名前"` / `--spots`（一覧表示）程度。まず対話UIを優先。

## 正直な制約・注意

- キーは既に多いので `P` 一段メニューにまとめる（トップキーの増加を1つに抑える）。
- 色パレットは10色。カテゴリが増え過ぎると色が回る（重複）。実用上は問題になりにくい。
- スポットが多いと左袖一覧が縦に伸びる（スクロールは将来対応）。

## 段階リリース

1. データ(2ファイル)＋読み書き、追加(a)、表示トグル(v)、地図重畳。
2. 一覧(l)：移動/削除。
3. カテゴリ管理(k)：新規/改名/色変更/削除(空のみ)。
